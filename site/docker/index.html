    Docker

    Docker cheat sheet

    Docker compose¶

    Create and start containers¶

    docker-compose up

    To run in background

    docker-compose up -d

    Stop containers¶

    docker-compose down

    ...and remove stopped containers

    docker-compose down --remove-orphans

    Dockerize flask app¶

    docker-entrypoint.sh¶

    For Linux If you want to access host from a docker container, you can find out more on how to do it here

    Flask¶

    Dockerfile

    FROM 3.11.3-slim

WORKDIR /app
COPY ./requirements.txt /app
COPY . ./app
RUN pip install -r requirements.txt

EXPOSE 5000
# `my_app/app.py` is where the flask app initiate
ENV FLASK_APP=my_app/app.py
# or `production`, `uat` etc
ENV FLASK_ENV=development

# if not using `docker-entrypoint` mentioned earlier
CMD ["flask", "run", "--host", "0.0.0.0"]
# or with gunicorn
# CMD ["gunicorn", "-b", ":5000", "my_app.app:app"]

    add python3 -m flask run to the end of docker-entrypoint.sh if using ENTRYPOINT

    Built the Docker image

    docker build -t my-flask-app .

    and run the container:

    docker run -p 5000:5000 my-flask-app
